Project Overview
把行政作業拆成模組
再交給 AI 串起來

這個專案不是單一 PDF 生成工具，而是一套以 Claude AI 為核心、可從手機遠端觸發本機流程的行政自動化系統，涵蓋文件產出、郵件準備、分類歸檔與執行紀錄。

Case Breakdown Slides 01 – 10
01
專案定位

AI 行政自動化系統

以日常行政任務為場景，將原本必須在電腦前逐步完成的作業轉為一句話或一個手機指令即可觸發。

遠端辦公流程 PDF 文件生成 Gmail 草稿建立 客戶歸檔 流程標準化 AI 協作
02
問題背景

痛點不是單一步驟
而是任務分散

原本的處理方式

  1. 開啟模板或舊檔案
  2. 手動填寫客戶資料
  3. 重新輸出 PDF
  4. 找到對應客戶資料夾
  5. 建立或撰寫寄信內容
  6. 確認附件與檔名

核心問題

每一步都不難，但分散、重複、需要切換工具，累積起來就是高頻低價值工作。

03
解法設計

用 Claude 當中控
把模組串成工作流

整體設計不是做單一腳本，而是讓 Claude 根據輸入內容決定要呼叫哪些模組，再透過本機執行完成整套任務。

使用者輸入需求
→
Claude 解析任務
→
觸發本機流程
→
產出文件 / 郵件 / 歸檔
Claude AI Telegram Bot Python Gmail API
04
遠端操控

透過 Telegram Bot
從手機遠端觸發

把 Telegram Bot 設計成遠端操作介面，外出時也可以透過手機下指令，讓本機電腦執行既定的行政任務。

輸入範例： 幫我建立王先生這個客戶的請款資料， 整理成 PDF，並準備寄件草稿。

重點不只是「可以遠端」，而是讓行政任務脫離必須坐在電腦前逐步操作的限制。

05
文件生成

自動產生報價單與請款單 PDF

文件生成模組接收客戶名稱、地址、品項與金額等資訊，自動計算欄位並輸出 PDF。這讓文件處理從「逐份編修」轉為「輸入資料後直接產出」。

模組輸入輸出價值
報價單生成客戶資料、品項、單價報價 PDF減少重複編修模板
請款單生成客戶資料、請款內容請款 PDF統一文件格式與欄位
檔案命名日期、客戶、案件資訊標準化檔名降低後續搜尋成本
06
寄信流程

串接 Gmail API
先完成寄件草稿

系統不一定直接寄出郵件，而是先建立 Gmail 草稿，讓使用者保留最後確認空間，同時省去重複填寫主旨、收件資訊與附件整理的時間。

  • 自動建立主旨與寄件內容
  • 對應附件與文件版本
  • 降低手動輸入錯誤
07
分類歸檔

不是只產生文件
而是建立案件紀錄

PDF 產出後依客戶與案件類型完成歸檔，讓文件不會停留在桌面或下載資料夾，而是回到可追蹤的結構中。

客戶資料
→
文件產出
→

→
保留紀錄
