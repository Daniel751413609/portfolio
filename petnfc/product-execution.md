# PetNFC 產品畫像與落地執行

## 產品一句話
PetNFC 是一個以 `NFC 吊牌 + 手機網頁 + LINE 通知` 組成的寵物身份識別平台，讓走失毛孩被掃到時，現場的人能立刻看到必要資訊，飼主能同步收到通知並快速接回。

## 先定義核心價值
- 對飼主：不是追蹤器，而是最低摩擦的「找回入口」
- 對路人：不用下載 App，掃一下就知道怎麼幫
- 對獸醫院 / 寵物店：可變成加值服務與通路商品

## 1. 誰在用？

### A. 飼主
這是主要付費者，也是資料維護者。

- 典型人物：
- 都市養狗 / 養貓族
- 已經熟悉 LINE，但不想再下載新 App
- 願意為安全感與身份識別付費

- 核心需求：
- 幫毛孩建立公開身份頁
- 萬一走失時能被快速聯絡
- 掃描發生時立刻收到通知
- 能自己更新聯絡方式、醫療提醒、個性備註

### B. 路人 / 拾獲者
這是最關鍵的啟動角色，決定找回流程是否順暢。

- 典型人物：
- 在路上、社區、公園遇到疑似走失毛孩的人
- 手機使用者，但不一定懂寵物晶片流程
- 願意幫忙，但只願意做 1 到 2 個動作

- 核心需求：
- 快速知道這隻寵物是不是有人照顧
- 快速聯絡飼主
- 不必註冊、不必安裝 App、不必理解複雜資訊

### C. 獸醫院 / 診所 / 寵物店
這是第二階段的重要通路角色，不是 MVP 主角，但很適合提早設計。

- 典型人物：
- 前台人員、獸醫助理、店員
- 想提供附加服務提升客單與黏著
- 需要低教學成本、低售後成本的產品

- 核心需求：
- 幫客戶快速啟用吊牌
- 協助更新基本資料
- 將 PetNFC 當成掛號、健檢、會員服務延伸

## 2. 他做什麼動作？

### A. 飼主主線流程
1. 購買吊牌
2. 用 LINE Login 登入
3. 綁定吊牌編號或感應啟用 NFC
4. 填寫寵物公開資料
5. 設定緊急聯絡方式與通知偏好
6. 將吊牌掛上毛孩

### B. 路人 / 拾獲者主線流程
1. 用手機掃 NFC
2. 開啟公開頁
3. 查看寵物名字、照片、性格、注意事項
4. 點擊按鈕
5. 直接打電話、傳 LINE、或送出「我找到牠了」表單

### C. 獸醫院 / 寵物店流程
1. 協助飼主挑選或購買吊牌
2. 引導飼主掃描啟用
3. 需要時代填基本資料
4. 在店內或診所作為會員加值服務

## 3. 完成後他看到什麼？

### A. 飼主完成後看到的結果
- 啟用成功頁
- 寵物公開頁預覽
- LINE 綁定成功訊息
- 未來每次被掃描時收到 LINE 通知

#### LINE 通知內容建議
- `你的毛孩「奶茶」剛剛在大安森林公園附近被掃描`
- 時間
- 掃描地點或地理描述
- 一鍵回撥 / 打開管理後台

### B. 路人完成後看到的結果
- 一個立即可讀的寵物資訊頁
- 明確 CTA：`聯絡飼主`、`我先帶去安全處`、`送去附近獸醫`
- 成功送出後的確認頁

#### 公開頁最小資訊集
- 寵物照片
- 名字
- 是否走失 / 需要聯絡飼主
- 個性描述：怕生、可接近、不能吃什麼
- 緊急聯絡按鈕
- 可選資訊：慢性病、用藥、晶片末四碼

### C. 獸醫院 / 寵物店完成後看到的結果
- 啟用完成頁
- 飼主資料已建立或已綁定
- 可回到門市流程的簡潔確認畫面

## 角色 x 動作 x 輸出 對照表

| 角色 | 動作 | 系統處理 | 最終看到什麼 |
|---|---|---|---|
| 飼主 | LINE 登入 | 建立帳號 | 進入後台 |
| 飼主 | 綁定吊牌 | 建立 tag 與 pet 關聯 | 啟用成功 |
| 飼主 | 填寫寵物資料 | 寫入資料庫 | 公開頁預覽 |
| 路人 | 掃 NFC | 打開公開頁、記錄 scan event | 寵物資訊頁 |
| 路人 | 按聯絡飼主 | 建立聯絡事件 | 成功提示頁 |
| 系統 | 接到 scan event | 發 LINE Bot 通知 | 飼主收到 LINE |
| 獸醫院 | 協助啟用 | 建立初始資料 | 啟用完成頁 |

## MVP 要先做什麼

### MVP 只做 3 條主線
1. 飼主可登入並建立寵物檔案
2. 路人掃 NFC 後可看到公開頁並聯絡飼主
3. 掃描發生後飼主會收到 LINE 通知

### 第一版先不要做
- GPS 即時追蹤
- 複雜社群協尋系統
- 保險串接
- AI 獸醫
- B2B 後台

## 落地執行拆解

### Phase 1：能用的最小閉環
- 頁面 1：飼主登入 / 啟用頁
- 頁面 2：飼主編輯寵物資料頁
- 頁面 3：NFC 掃描公開頁
- 頁面 4：聯絡成功頁
- 系統 1：LINE Login
- 系統 2：LINE Bot 掃描通知
- 系統 3：scan event 紀錄

### Phase 2：提高找回率
- 增加「我找到牠了」表單
- 增加位置資訊與時間軸
- 增加多聯絡方式
- 增加走失狀態切換

### Phase 3：擴通路與商業化
- 診所 / 寵物店代啟用流程
- 門市專屬推薦碼
- Premium 訂閱
- 品牌聯名與 B2B 報表

## 頁面與功能對應

### 1. 飼主啟用頁
- 使用者：飼主
- 動作：LINE 登入、輸入或掃描吊牌 ID
- 完成後：看到啟用成功，進入資料編輯

### 2. 飼主後台編輯頁
- 使用者：飼主
- 動作：填寫毛孩照片、名字、品種、年齡、個性、醫療提醒、聯絡方式
- 完成後：資料更新成功，可預覽公開頁

### 3. NFC 公開頁
- 使用者：路人 / 拾獲者
- 動作：掃 NFC 後自動進站
- 完成後：看到寵物資料與聯絡按鈕

### 4. 聯絡行動頁
- 使用者：路人 / 拾獲者
- 動作：打電話、傳訊息、送出發現地點
- 完成後：看到「已通知飼主」

### 5. LINE 通知
- 使用者：飼主
- 動作：被動接收
- 完成後：看到掃描時間、位置、聯絡入口

## 最重要的產品判斷

### 這不是「管理平台」優先
第一優先是讓陌生人願意幫你完成第一個動作，也就是 `掃一下就能幫忙聯絡`。

### 這不是「追蹤器」優先
PetNFC 在第一階段賣的是 `身份識別 + 找回效率 + 飼主安心感`，不是硬體定位能力。

### 這不是「給獸醫院用的 SaaS」起手
獸醫院 / 寵物店是通路放大器，但第一個產品閉環一定是 `飼主買單、路人幫忙、LINE 通知飼主`。

## 一句話版本
PetNFC 的第一版產品畫像很清楚：

`飼主建立毛孩檔案 -> 路人掃 NFC -> 公開頁立即顯示 -> 系統同步發 LINE 給飼主 -> 飼主快速接回。`
